## 1. struct（结构体）是什么

结构体就是把多个字段组合成一个新类型：

```go
type User struct {
	ID   int
	Name string
}
```

创建方式：

```go
u1 := User{ID: 1, Name: "Alice"} // 推荐：字段名写清楚
u2 := User{}                     // 全零值
_, _ = u1, u2
```

---

## 2. 方法（method）是什么

方法就是“绑定在某个类型上的函数”：

```go
func (u User) DisplayName() string { ... }
```

### 2.1 值接收者 vs 指针接收者

你只需要先掌握两个规则：

- **想修改接收者本身**：用指针接收者 `func (u *User) SetName(...)`
- **类型比较大/避免拷贝**：也用指针接收者

如果方法只是读数据，值接收者也可以。

注意：当你写了指针接收者的方法，Go 也允许你用值来调用（编译器会自动取地址），反之亦然有一些限制，但初学阶段记住“修改就用指针”基本够用。

---

## 3. interface（接口）是什么

接口是一组方法的集合：

```go
type Stringer interface {
	String() string
}
```

### 3.1 Go 的接口是“隐式实现”

Go 里不需要写 `implements`。

只要一个类型拥有接口要求的方法，它就自动实现该接口。

这也是 Go 很推崇的小接口风格的原因：接口越小，越容易复用。

---

## 4. embedding（组合/嵌入）是什么

嵌入是一种“用组合代替继承”的做法：

```go
type Base struct {
	ID int
}

type User struct {
	Base
	Name string
}
```

你可以直接通过 `u.ID` 访问嵌入字段（像“提升”了一样），但本质仍是组合。

---

## 5. fmt.Stringer：一个最常见的接口例子

标准库里经常用 `fmt.Stringer`：

```go
type Stringer interface {
	String() string
}
```

实现它后：
- `fmt.Println(x)` 会更漂亮
- 调试输出更清晰

## 6. 初学者常见坑

1. 用值接收者写“修改方法”，导致修改不生效
2. 以为接口需要显式声明实现
3. 把 embedding 当继承使用（Go 更推荐明确组合与委托）

## 1. 类型断言（Type Assertion）

类型断言用于从接口值中提取具体类型。Go 是静态类型语言，接口可以存储任意类型，但你需要知道它实际是什么类型才能操作它。

### 1.1 基本语法

```go
value, ok := interfaceValue.(Type)
```

- `interfaceValue`：接口类型的变量
- `Type`：你想要断言的类型
- `value`：断言成功后的具体类型值
- `ok`：布尔值，表示断言是否成功

```go
var i interface{} = "Hello, Go!"

// 尝试断言为 string
s, ok := i.(string)
if ok {
	fmt.Println("断言成功:", s) // 断言成功: Hello, Go!
}

// 尝试断言为 int
n, ok := i.(int)
if ok {
	fmt.Println("断言成功:", n)
} else {
	fmt.Println("断言失败") // 断言失败
}
```

> ⚠️ **注意**：**永远优先用 `value, ok := i.(T)` 这种带 ok 的形式**。它会安全返回 `false` 而不是崩溃。只有你 100% 确定类型对得上时才用单返回值形式，否则就是埋雷。

### 1.2 另一种形式（会 panic）

```go
value := interfaceValue.(Type) // 不返回 ok，失败直接 panic
```

**不推荐**，除非你 100% 确定类型正确：

```go
var i interface{} = "Hello, Go!"
s := i.(string)  // OK
// n := i.(int)  // panic: interface conversion: interface {} is string, not int
```

### 1.2.1 实战对照：带 ok 形式 vs 单值形式（panic）

> ⚠️ **注意（教材强调的对照）**：同一个类型断言，两种写法行为天差地别。务必用带 `ok` 的安全形式，单值形式只在你 100% 确定类型时再用。

```go
// 来自 typesx.go
func TypeAssertString(val interface{}) (string, bool) {
	s, ok := val.(string) // 安全：失败返回 ok=false，不 panic
	return s, ok
}

func MustTypeAssertString(val interface{}) string {
	return val.(string) // 危险：类型不对直接 panic
}
```

对应测试见：[typesx_typesx_test.go](../16-tests/typesx_typesx_test.go)（`TestMustTypeAssertString` 验证了 `MustTypeAssertString(123)` 会 panic，而 `TypeAssertString("hello")` 安全返回）。

> ⚠️ **注意**：`MustTypeAssertString` 这种命名（Must 前缀）在 Go 生态里是约定俗成的"不满足就 panic"信号（类似 `template.Must`）。调用前你必须自己保证类型正确，否则直接崩。生产代码优先用 `TypeAssertString` 这种返回 `(value, ok)` 的形式，用 `if !ok` 优雅兜底。

### 1.3 Type Switch（类型开关）

当接口可能存储多种类型时，用 type switch：

```go
// 对应 typesx.go 的 TypeSwitch
func TypeSwitch(val interface{}) string {
	switch val.(type) {
	case int:
		return "int"
	case string:
		return "string"
	case float64:
		return "float64"
	default:
		return "unknown"
	}
}
```

对应 `typesx.go` 的 `TypeAssertString` 也是同样思路：

```go
func TypeAssertString(val interface{}) (string, bool) {
	s, ok := val.(string)
	return s, ok
}
```

### 1.4 为什么需要类型断言

Go 没有泛型的早期版本（Go 1.18 之前），很多函数返回 `interface{}`，你必须用类型断言来使用返回值。

即使有了泛型，类型断言在以下场景仍然有用：
- 处理 `any` 类型的配置值
- 解析 JSON 等动态数据
- 实现插件系统

> ⚠️ **注意**：`interface{}` 现在写作 `any`（Go 1.18+ 的别名），两者完全等价。类型断言本质是把“盒子里的动态值”重新固定成静态类型，断言失败说明**运行时实际类型和你假设的不一致**——这属于逻辑 bug，应当用 `ok` 分支优雅处理，而不是靠 panic 兜底。

### 1.5 类型断言 vs 类型转换

| 操作 | 语法 | 用途 |
|------|------|------|
| 类型断言 | `v, ok := i.(Type)` | 从接口提取具体类型 |
| 类型转换 | `v := Type(expr)` | 同一底层类型间转换 |

```go
// 类型断言
var i interface{} = 42
n, ok := i.(int) // 从 interface{} 提取 int

// 类型转换
var x int32 = 42
y := int64(x) // int32 转 int64
```

## 1.6 空标识符 `_` 的三种用法

空标识符 `_` 用于"声明但不使用"的场景，避免"未使用变量"编译错误：

```go
// 来自 typesx.go
func DiscardValue() string {
	_, v := MultiReturn() // ① 忽略某个返回值
	return v
}

func MultiReturn() (string, string) {
	return "ignored", "kept"
}
```

对应测试见：[typesx_typesx_test.go](../16-tests/typesx_typesx_test.go)（`TestDiscardValue`）。

> ⚠️ **注意（三种常见用法，教材《Learning Go》归纳）**：
> 1. **忽略某个返回值**：`_, v := m[k]`（只要 value，丢弃 ok）
> 2. **导入副作用**：`import _ "pkg"`（只执行包的 `init()`，不引用其任何标识符，常见于注册驱动如 `sql.Register`）
> 3. **for-range 忽略索引或值**：`for _, v := range xs`（只要值，不要下标）
>
> `_` 本身不是普通变量，不能对它取地址或再次赋值。它本质是编译器允许的"丢弃槽"。

## 1.7 strings.Builder：高效字符串拼接

Go 字符串不可变，循环用 `+` 拼接是 O(N²) 的隐患：

```go
// 来自 typesx.go
func JoinWithBuilder(parts []string) string {
	var b strings.Builder
	for _, p := range parts {
		b.WriteString(p) // 复用内部 []byte 缓冲区
	}
	return b.String()
}
```

对应测试见：[typesx_typesx_test.go](../16-tests/typesx_typesx_test.go)（`TestJoinWithBuilder` 覆盖空/单/多/含空格四组用例）。

> ⚠️ **注意（性能陷阱）**：`s += piece` 每次都分配新内存并复制整个字符串，循环 N 次整体是 O(N²)。`strings.Builder` 复用内部 `[]byte` 缓冲区，循环拼接是 O(N)，大数据量下差异巨大。教材《Learning Go》也强调：拼接多个字符串优先用 `strings.Builder`（或 `strings.Join` 一次性拼接切片）。另外 `bytes.Buffer` 也能用，但 `strings.Builder` 更轻量、专为字符串设计、不可被并发读写。

## 2. 继承？Go 用组合（Composition）

Go 没有类和继承，而是用 **struct embedding（结构体嵌入）** 实现代码复用。

### 2.1 基本嵌入

```go
type Animal struct {
	Name string
}

func (a Animal) Speak() string {
	return a.Name + " says hello"
}

// Dog 嵌入 Animal
type Dog struct {
	Animal // 嵌入，不是字段
	Breed string
}
```

`typesx.go` 里的 `Admin` 嵌入 `User` 是同样模式：

```go
type Admin struct {
	User  // 嵌入 User
	Level int
}
```

### 2.2 字段提升（Field Promotion）

嵌入的字段和方法会被"提升"到外层类型，可以直接访问：

```go
admin := Admin{
	User:  User{ID: 1, Name: "Alice"},
	Level: 1,
}
fmt.Println(admin.Name) // Alice（字段提升，等价于 admin.User.Name）
fmt.Println(admin.Level) // 1
```

> ⚠️ **注意**：提升只是“语法糖”，`admin.Name` 实际访问的是 `admin.User.Name`。`Admin` 仍然**不是** `User` 类型，不能赋值给 `User` 变量。Go 没有 is-a 的继承链，只有 has-a 的组合。对应 Rust：相当于 `struct Dog { animal: Animal, ... }` 再手动转发，Go 只是把转发省掉了。

### 2.3 方法重写（Override）

嵌入后可以重写父类型的方法（实际是“外层方法遮蔽内层方法”）：

```go
type Animal struct {
	Name string
}

func (a Animal) Speak() string {
	return a.Name + " says hello"
}

type Dog struct {
	Animal
	Breed string
}

// 重写 Speak 方法
func (d Dog) Speak() string {
	return d.Name + " says woof!"
}
```

```go
dog := Dog{Animal: Animal{Name: "Buddy"}, Breed: "Golden"}
fmt.Println(dog.Speak())        // Buddy says woof!（调用 Dog 的方法）
fmt.Println(dog.Animal.Speak()) // Buddy says hello（调用 Animal 的方法）
```

### 2.4 多层嵌入

```go
type A struct{}

func (a A) Hello() string { return "A" }

type B struct{ A }

func (b B) Hello() string { return "B" }

type C struct{ B }
```

```go
c := C{}
fmt.Println(c.Hello())      // B（调用 B 的方法，最近的外层优先）
fmt.Println(c.B.Hello())    // B
fmt.Println(c.B.A.Hello())  // A
```

> ⚠️ **注意**：方法查找遵循“**外层优先 / 深度优先**”规则：同名方法，越靠近外层（越晚嵌入）越优先；想调用被遮蔽的内层方法，必须显式写出完整路径（如 `c.B.A.Hello()`）。

### 2.5 组合 vs 继承

| 特性 | 经典继承 | Go 组合 |
|------|---------|---------|
| 关系 | "是一个"（is-a） | "有一个"（has-a） |
| 代码复用 | 通过继承 | 通过嵌入 |
| 多态 | 方法重写 | 接口实现 |
| 灵活性 | 继承关系固定 | 运行时可组合 |

**Go 的哲学：**
> "Accept interfaces, return structs"
> "Do not communicate by sharing memory; instead, share memory by communicating"

## 3. 接口与组合的配合

### 3.1 小接口原则

Go 推崇小接口，一个接口只定义一个方法：

```go
// 标准库的小接口
type Reader interface {
	Read(p []byte) (n int, err error)
}

type Writer interface {
	Write(p []byte) (n int, err error)
}

type Stringer interface {
	String() string
}
```

> ⚠️ **注意**：小接口容易“隐式实现”。因为实现成本极低（只要加一个方法），任何类型都能顺手满足 `Stringer` / `Reader`。这正是 Go 标准库大量使用单方法接口的原因：`io.Reader`、`fmt.Stringer` 几乎可以被任何类型满足。

### 3.2 接口组合

```go
// 对应 typesx.go 的 ReadWriter
type ReadWriter interface {
	Reader
	Writer
}
```

`typesx.go` 正是这么做的：

```go
type Reader interface {
	Read() string
}

type Writer interface {
	Write(string)
}

type ReadWriter interface {
	Reader
	Writer
}
```

### 3.3 通过组合实现接口

```go
type Logger struct {
	prefix string
}

func (l Logger) Log(msg string) string {
	return l.prefix + ": " + msg
}

type Service struct {
	Logger // 嵌入 Logger
	name   string
}

// Service 自动拥有 Log 方法，可以当作有 Log 方法的接口使用
```

## 4. 常见陷阱

### 4.1 嵌入不是继承

```go
type Animal struct {
	Name string
}

type Dog struct {
	Animal
}

func main() {
	dog := Dog{Animal: Animal{Name: "Buddy"}}

	// Dog 不是 Animal
	// var a Animal = dog  // 编译错误

	// 但 Dog 有 Animal 的所有字段和方法
	fmt.Println(dog.Name) // Buddy
}
```

### 4.2 嵌入指针

```go
type Animal struct {
	Name string
}

type Dog struct {
	*Animal // 嵌入指针
	Breed string
}

func main() {
	dog := &Dog{
		Animal: &Animal{Name: "Buddy"},
		Breed:  "Golden",
	}

	fmt.Println(dog.Name) // Buddy
}
```

> ⚠️ **注意**：嵌入**指针**时，如果指针是 nil（比如 `Dog{}` 没初始化 `Animal`），访问 `dog.Name` 会 panic（nil 指针解引用）。嵌入值类型则没有这个问题（零值结构体字段可用）。

### 4.3 名字冲突

如果多个嵌入类型有同名字段或方法，必须显式指定：

```go
type A struct{ Name string }
type B struct{ Name string }

type C struct {
	A
	B
}

func main() {
	c := C{
		A: A{Name: "A"},
		B: B{Name: "B"},
	}

	// fmt.Println(c.Name)  // 编译错误：ambiguous
	fmt.Println(c.A.Name) // A
	fmt.Println(c.B.Name) // B
}
```

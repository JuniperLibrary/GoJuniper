## 0. 先建立一个“脑模型”

你可以把 Go 里的“名字”分成两大类：

- **变量（var / :=）**：运行期存在、可变、通常“有内存位置”
- **常量（const）**：编译期确定、不可变、很多时候只是“编译器替换”，不一定有内存位置

作用域就是：这些名字在什么范围内可见。

初始化就是：这些名字的值是按什么顺序被计算出来的。

---

## 1. 变量（var）是什么

变量（variable）就是“有名字的可变值”。Go 用 `var` 声明变量：

```go
var age int = 18
var name string = "Alice"
```

### 1.1 类型可写可不写（类型推断）

```go
var age = 18       // 推断为 int
var name = "Alice" // 推断为 string
```

### 1.2 零值（zero value）

Go 的变量只要声明了，一定有值（默认零值），不会像某些语言那样是“未初始化”。

常见零值：
- `int`/`float64`：`0` / `0.0`
- `bool`：`false`
- `string`：`""`
- 指针/切片/映射/channel/函数/interface：`nil`
- struct：每个字段都是零值（整体也有一个“全零”的状态）

例子：

```go
var a int             // 0
var b bool            // false
var s string          // ""
var p *int            // nil
var xs []int          // nil
var m map[string]int  // nil
```

零值很重要的原因：
- 你可以写出更少的初始化代码
- 很多类型零值就能直接用（例如 `var mu sync.Mutex`）

## 2. 变量声明的几种写法（你必须会）

### 2.1 单个 var

```go
var x int
var y int = 10
var z = 20 // 推断类型
```

### 2.2 var 组（常见于包级变量与结构化声明）

```go
var (
	host = "127.0.0.1"
	port = 8080
)
```

### 2.3 多重赋值（交换变量/同时返回）

```go
a, b := 1, 2
a, b = b, a
```

## 3. 短变量声明（:=）

在函数内部，你最常用的是 `:=`：

```go
func demo() {
	x := 10
	name := "Alice"
	_, _ = x, name
}
```

### 3.1 `:=` 的限制

- 只能在**函数内部**使用（不能在包级/全局使用）
- 左侧至少有一个变量是“新声明”的（否则会报错）

```go
func demo() {
	x := 10
	x := 20 // 编译错误：no new variables on left side of :=
	_ = x
}
```

但这种是允许的（因为 `y` 是新变量，`x` 是重新赋值）：

```go
func demo() {
	x := 10
	x, y := 20, 30 // OK
	_, _ = x, y
}
```

### 3.2 := 最容易踩的坑：遮蔽（shadowing）

你以为是在“给外层变量赋值”，其实声明了一个新变量，导致外层变量没变。

```go
func demo() error {
	var err error
	if true {
		err := doSomething() // 新变量 err，遮蔽了外层 err
		_ = err
	}
	return err // 返回外层 err，可能还是 nil
}
```

修复方法：把 `:=` 改成 `=`（前提是外层已声明）。

## 4. 作用域（scope）到底怎么理解

作用域就是“这个名字在代码的哪一段可见”。

Go 常见的作用域层级：
- **包级作用域**：在 `package xxx` 下、函数外声明的
- **函数级作用域**：函数体内声明的
- **块级作用域**：`{ ... }` 内声明的（例如 `if`/`for`/`switch`）

### 4.1 块作用域示例

```go
func demo() {
	x := 1
	{
		x := 2 // 新变量，遮蔽外层 x
		_ = x
	}
	_ = x // 这里仍然是 1
}
```

### 4.2 if/for 里常见的“局部声明”

```go
if v := compute(); v > 0 {
	// v 只在 if 的这个块里可见
}
```

```go
for i := 0; i < 10; i++ {
	// i 只在 for 循环体内可见
}
```

## 5. 全局变量（包级变量）

“全局变量”在 Go 里通常指**包级变量**（package-level variable），写在函数外面：

```go
package p

var GlobalCount int
```

### 5.1 包级变量的特点

- 生命周期：程序启动到退出一直存在
- 可见性（是否能被其它包使用）：看首字母大小写
  - `GlobalCount`（大写）：导出，可被其它包导入使用
  - `globalCount`（小写）：不导出，仅包内使用

### 5.2 什么时候用包级变量（学习阶段 vs 工程阶段）

学习阶段你会经常见到包级变量，但工程里要谨慎：

更推荐的包级变量用法：
- 只读的固定值（例如一些默认值）
- 错误变量：`var ErrX = errors.New("...")`
- 只读的查表数据（map/slice），且不在运行中修改

不推荐的包级变量用法：
- 可变的全局状态（并发不安全、测试困难、依赖隐式）

工程上的常见替代：
- 把状态放进 struct 里，通过构造函数创建实例（让依赖显式）
- 通过参数传递依赖（函数式/依赖注入）

## 6. 常量（const）

常量（constant）是“编译期就确定”的值，不能修改：

```go
const Pi = 3.14159
const AppName = "GoJuniper"
```

### 6.1 const 与 var 的关键区别

- `var`：运行期有内存地址，可变
- `const`：编译期替换，不可变，通常没有“可取地址”的概念

因此，`const` 常用于：
- 枚举（状态码、选项）
- 固定配置（默认值、版本号）
- 位运算标志（flag）

### 6.2 常量表达式的限制

常量必须由“常量表达式”构成，不能调用运行期函数：

```go
const A = 1 + 2     // OK
// const B = time.Now() // 编译错误：不是常量
```

### 6.3 未类型常量（untyped constant）

这是 Go 的强项：常量可以不绑定具体类型，在使用时自动适配。

```go
const x = 10 // untyped

var a int = x
var b int64 = x
var c float64 = x
```

对比：变量会固定类型，不能随意赋给别的整数类型。

```go
var x = 10 // x 是 int
// var a int64 = x // 编译错误
```

### 6.4 iota（枚举）

```go
const (
	StatusInit = iota // 0
	StatusRunning     // 1
	StatusDone        // 2
)
```

常见写法：跳过 0（让默认值更安全）

```go
const (
	_ = iota
	StatusRunning
	StatusDone
)
```

### 6.5 iota + 位运算（flags）

```go
const (
	FlagRead  = 1 << iota // 1
	FlagWrite             // 2
	FlagExec              // 4
)
```

## 7. 声明 vs 赋值（必须分清）

- 声明（declare）：第一次引入名字
  - `var x int`
  - `x := 1`
- 赋值（assign）：给已存在的变量改值
  - `x = 2`

最常见错误：你想“赋值”，结果写了 `:=`，导致遮蔽或编译错误。

## 8. Go 的函数（func）怎么写：几个必须会的例子

### 8.1 最基础：输入参数 + 返回值

```go
func Sum(a, b int) int {
	return a + b
}
```

### 8.2 多返回值：Go 的标配风格

Go 很常见的写法是“结果 + 是否成功/错误”：

```go
func Max(xs []int) (int, bool) {
	if len(xs) == 0 {
		return 0, false
	}
	m := xs[0]
	for _, v := range xs[1:] {
		if v > m {
			m = v
		}
	}
	return m, true
}
```

这也是你在标准库里经常看到的模式（例如 map 的读取 `v, ok := m[k]`）。

### 8.3 命名返回值（新手先谨慎使用）

命名返回值会把返回值当作函数内的变量：

```go
func Divide(a, b int) (q int, ok bool) {
	if b == 0 {
		return 0, false
	}
	return a / b, true
}
```

初学阶段建议你优先写“显式 return 值”，减少误读。

### 8.4 变参函数（...）

```go
func SumAll(xs ...int) int {
	total := 0
	for _, v := range xs {
		total += v
	}
	return total
}
```

调用时：

```go
_ = SumAll(1, 2, 3)
xs := []int{1, 2, 3}
_ = SumAll(xs...) // slice 展开
```

### 8.5 defer：退出前执行（常用于关闭资源）

```go
func demo() error {
	f, err := os.Open("a.txt")
	if err != nil {
		return err
	}
	defer f.Close()
	return nil
}
```

记住一句话：defer 通常用于“打开资源后立即写上关闭”，避免忘记释放。

## 9. Go 的包（package）与引用（import）：从新手到能跑起来

### 9.1 一个 Go 文件的最小结构

```go
package main

import "fmt"

func main() {
	fmt.Println("hello")
}
```

### 9.2 import 的两种写法

单行：

```go
import "fmt"
```

多行（更常见）：

```go
import (
	"fmt"
	"strings"
)
```

### 9.3 引用你自己仓库里的包：用模块路径（go.mod 的 module）

本仓库的 module 是 `gojuniper`，所以导入路径形如：

```go
import "gojuniper/internal/basics"
```

然后就可以调用导出的标识符（大写开头）：

```go
fmt.Println(basics.Sum(1, 2))
```

你可以直接看现成的可运行例子：[cmd/hello/main.go](file:///e:/dingchuan/GoJuniper/cmd/hello/main.go#L1-L15)

### 9.4 import 别名（alias）

当包名冲突或你想让代码更清楚时可以用别名：

```go
import b "gojuniper/internal/basics"

func main() {
	_ = b.Sum(1, 2)
}
```

### 9.5 internal 目录的特殊规则（你现在就用上了）

Go 里 `internal/` 是“限制导入范围”的机制：
- 只有位于 `internal` 父目录范围内的代码才能导入它
- 这能保证这些学习包只在本仓库内部使用，避免被别的项目直接依赖

## 10. 初始化顺序：包级变量与 init()

你可以用这个“足够正确的简化规则”理解 Go 初始化：

1. 先初始化依赖包（import 的包）
2. 再初始化当前包的：
   - 包级变量（按依赖关系决定顺序）
   - `init()` 函数（同一个包可以有多个 init，依次执行）
3. 最后进入 `main.main()`

### 8.1 init() 什么时候用

学习阶段会遇到 `init()`，但工程中不要滥用，因为它会让初始化变得“隐式”。

更推荐显式构造：

```go
func NewService(...) *Service { ... }
```

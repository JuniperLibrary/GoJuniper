## 1. 算术运算符

| 运算符 | 描述 | 示例 |
|--------|------|------|
| `+` | 相加 | `10 + 5` → `15` |
| `-` | 相减 | `10 - 5` → `5` |
| `*` | 相乘 | `10 * 5` → `50` |
| `/` | 相除 | `10 / 3` → `3`（整数除法） |
| `%` | 求余 | `10 % 3` → `1` |
| `++` | 自增 | `a++`（只能后置，不能前置） |
| `--` | 自减 | `a--`（只能后置，不能前置） |

### 1.1 整数除法 vs 浮点除法

```go
fmt.Println(10 / 3)    // 3
fmt.Println(-10 / 3)   // -3（向零截断）
fmt.Println(10.0 / 3)  // 3.3333333333333335
fmt.Println(float64(10) / 3)  // 3.3333333333333335
```

> ⚠️ **注意（重点）**：整数除法 `/` 是**向零截断**（`-10/3 == -3`，不是 `-4`）。这是 Go 和某些语言（向负无穷取整）的差异。要浮点结果，至少一个操作数是浮点（`10.0/3` 或 `float64(10)/3`）。`%` 同样只用于整数（取余），浮点取模用 `math.Mod`。

### 1.2 自增自减的特殊性

Go 的 `++` 和 `--` 只能作为语句，不能作为表达式：

```go
a := 5
a++      // OK：a 变成 6
// b := a++  // 编译错误：不能用作表达式
// c := a-- + 1  // 编译错误
```

> ⚠️ **注意**：Go **没有** `++a` / `--a`（前置），只有后置 `a++`。且 `a++` 是语句不是表达式，不能 `b := a++` 或用于复杂表达式。这是刻意删掉 C 的"前置/后置+表达式"歧义，让语义唯一。

## 2. 关系运算符

| 运算符 | 描述 | 示例 |
|--------|------|------|
| `==` | 等于 | `10 == 10` → `true` |
| `!=` | 不等于 | `10 != 5` → `true` |
| `>` | 大于 | `10 > 5` → `true` |
| `<` | 小于 | `10 < 5` → `false` |
| `>=` | 大于等于 | `10 >= 10` → `true` |
| `<=` | 小于等于 | `10 <= 5` → `false` |

### 2.1 比较的限制

- 不同类型不能直接比较：`int` 和 `int64` 不能用 `==`
- 切片、map、函数不能用 `==`（只能和 `nil` 比较）
- struct 可以比较（如果所有字段都可以比较）

```go
var a int = 10
var b int64 = 10
// fmt.Println(a == b)  // 编译错误：类型不匹配
fmt.Println(a == int(b))  // true（显式转换后）

s1 := []int{1, 2, 3}
s2 := []int{1, 2, 3}
// fmt.Println(s1 == s2)  // 编译错误
// 要比较切片，需要手动遍历或用 reflect.DeepEqual
```

> ⚠️ **注意（重点）**：**切片、map、函数之间不能用 `==` 比较**（编译错误），只能和 `nil` 比。想比较内容要用 `reflect.DeepEqual` 或手动遍历。而 struct 只要字段都可比较就能用 `==`（含数组字段；但含 slice/map 字段的 struct 不能用 `==`）。对比 Java：Java 的对象 `==` 比较的是引用（地址），内容比较需用 `equals()`；`ArrayList` 默认 `equals` 会比较内容，但 Go 没有 `equals` 方法约定——Go 这里更严格（基本不允许 `==` 比较引用类型）。

## 3. 逻辑运算符

| 运算符 | 描述 | 示例 |
|--------|------|------|
| `&&` | 逻辑与（AND） | `true && false` → `false` |
| `||` | 逻辑或（OR） | `true \|\| false` → `true` |
| `!` | 逻辑非（NOT） | `!true` → `false` |

### 3.1 短路求值

```go
func sideEffect() bool {
    fmt.Println("执行了")
    return true
}
if false && sideEffect() { }  // sideEffect() 不会被调用
if true || sideEffect() { }   // sideEffect() 不会被调用
```

## 4. 位运算符

| 运算符 | 描述 | 示例 |
|--------|------|------|
| `&` | 按位与 | `0101 & 0011` → `0001` |
| `|` | 按位或 | `0101 \| 0011` → `0111` |
| `^` | 按位异或 | `0101 ^ 0011` → `0110` |
| `&^` | 按位清除（AND NOT） | `0101 &^ 0011` → `0100` |
| `<<` | 左移 | `1 << 3` → `8` |
| `>>` | 右移 | `8 >> 2` → `2` |

> ⚠️ **注意（Go 特有）**：`&^` 是 Go 的"按位清除"（AND NOT）：`a &^ b` 把 b 为 1 的位清 0。这是 Go 独有运算符，其他语言一般没有。移除权限位常用 `perm &^= Exec`。

### 4.1 位运算的实际用途

```go
const (
    Read    = 1 << iota // 0001 = 1
    Write               // 0010 = 2
    Exec                // 0100 = 4
)

hasRead := (permission & Read) != 0   // 检测：用 != 0，不是 ==
permission |= Write                  // 添加
permission &^= Exec                   // 移除

if n&1 == 0 { fmt.Println("偶数") }    // 奇偶判断
n << 3   // 等价于 n * 8
n >> 2   // 等价于 n / 4
```

> ⚠️ **注意**：位标志检测必须写 `perm & Flag != 0`，**不能**写 `perm == Flag`（后者要求其他位都为 0）。`1 << iota` 生成互不重叠的位，才能 `|` 组合、`&` 检测、`&^` 移除。详见 [常量与变量与作用域与初始化.md](./常量与变量与作用域与初始化.md) 的 iota 小节。

## 6. 运算符优先级

优先级从高到低：

| 优先级 | 运算符 |
|--------|--------|
| 5 | `* / % << >> & &^` |
| 4 | `+ - \| ^` |
| 3 | `== != < <= > >=` |
| 2 | `&&` |
| 1 | `\|\|` |

**建议**：不确定优先级时，用括号明确：

```go
result := a + (b * c)  // 推荐，清晰
```

> ⚠️ **注意**：`&&` 优先级高于 `||`。复杂布尔表达式（如 `a || b && c`）容易读错，强烈建议加括号。`a || b && c` 实际是 `a || (b && c)`。

## 7. if 条件语句

### 7.1 基本语法

```go
if condition {
} else {
}
```

### 7.2 if-else if-else

```go
if score >= 90 {
    fmt.Println("优秀")
} else if score >= 80 {
    fmt.Println("良好")
} else if score >= 60 {
    fmt.Println("及格")
} else {
    fmt.Println("不及格")
}
```

### 7.3 if 带初始化语句

```go
if err := doSomething(); err != nil {
    fmt.Println("出错了:", err)
}
// 等价于
err := doSomething()
if err != nil { fmt.Println("出错了:", err) }
```

**常见用途**：

```go
if f, err := os.Open("file.txt"); err != nil {
    fmt.Println("打开文件失败:", err)
} else {
    defer f.Close()
    // 使用文件
}

if s, ok := i.(string); ok {  // 类型断言
    fmt.Println("字符串:", s)
}
```

### 7.4 条件必须是布尔类型

```go
x := 5
// if x { }  // 编译错误：x 不是 bool 类型
if x != 0 {
    fmt.Println("非零")
}
```

> ⚠️ **注意（Java 对照）**：Go 的 `if` 条件**必须是布尔表达式**，不能写 `if x {}`（x 是非零整数也不行）。这点和 Java `if (x != 0)` 一致，但和 C（`if (x)` 把非零当真）不同。强类型约束，避免"非零即真"的隐式转换 bug。

> ⚠️ **注意**：`if v, ok := i.(type); ok` 是类型断言的"逗号 ok"写法，失败时 `ok==false` 且 `v` 为零值，**不会 panic**。而直接 `v := i.(string)` 失败时**会 panic**——不确定类型时永远用 `ok` 形式。

## 8. switch 多分支

### 8.1 基本语法

```go
switch expression {
case value1:
case value2:
default:
}
```

### 8.2 无条件 switch（代替 if-else if）

```go
switch {
case score >= 90:
    fmt.Println("优秀")
case score >= 80:
    fmt.Println("良好")
case score >= 60:
    fmt.Println("及格")
default:
    fmt.Println("不及格")
}
```

### 8.3 多值匹配

```go
switch day {
case "Saturday", "Sunday":
    fmt.Println("周末")
case "Monday", "Tuesday", "Wednesday", "Thursday", "Friday":
    fmt.Println("工作日")
default:
    fmt.Println("无效")
}
```

### 8.4 类型 switch

```go
func describe(i interface{}) {
    switch v := i.(type) {
    case int:
        fmt.Printf("整数: %d\n", v)
    case string:
        fmt.Printf("字符串: %q\n", v)
    case bool:
        fmt.Printf("布尔: %v\n", v)
    default:
        fmt.Printf("未知类型: %T\n", v)
    }
}
```

### 8.5 fallthrough（穿透）

Go 的 switch 默认不会穿透到下一个 case（和 C/Java 不同）。如果需要穿透，用 `fallthrough`：

```go
switch x {
case 1:
    fmt.Println("一")
    fallthrough
case 2:
    fmt.Println("二")  // 如果 x=1，这里也会执行
case 3:
    fmt.Println("三")
}
```

> ⚠️ **注意（重点）**：Go 的 `switch` **默认不穿透**——匹配到 case 执行完直接跳出，不需要写 `break`（和 C 相反，Java 传统 `switch` 也需 break 否则 fall-through）。需要穿透才写 `fallthrough`，且它**只穿透到下一个 case**，不能跳多个。另外"无条件 switch"（裸 `switch {}`）是代替 if-else if 链的惯用法，更清晰。

### 8.6 switch 中的初始化语句

```go
switch os := runtime.GOOS; os {
case "linux":
    fmt.Println("Linux 系统")
case "darwin":
    fmt.Println("macOS 系统")
case "windows":
    fmt.Println("Windows 系统")
default:
    fmt.Println("其他系统:", os)
}
```

## 9. for 循环

Go 只有 `for` 一种循环，但可以实现 while、do-while 的效果。

### 9.1 基本 for 循环

```go
for i := 0; i < 10; i++ {
    fmt.Println(i)
}
```

### 9.2 while 风格

```go
n := 1
for n < 1000 {
    n *= 2
}
```

### 9.3 无限循环

```go
for {
    if done {
        break
    }
}
```

> ⚠️ **注意**：Go **只有 `for` 一种循环关键字**，没有 `while`/`do-while`。`for condition {}` 就是 while，`for {}` 就是无限循环。这是 Go"少关键字"哲学的体现。配合 `break`/`continue`/`label` 足够表达所有循环模式。

## 10. range 迭代

`range` 用于遍历切片、数组、map、字符串、channel。

### 10.1 遍历切片/数组

```go
nums := []int{10, 20, 30}
for _, v := range nums { fmt.Println(v) }       // 只要值
for i, v := range nums { fmt.Printf("%d:%d\n", i, v) }  // 索引+值
for i := range nums { fmt.Println(i) }          // 只要索引
```

### 10.2 遍历字符串

```go
for i, r := range "Hello, 你好" {
    fmt.Printf("索引 %d: %c\n", i, r)
}
// 索引 0:H 1:e 2:l 3:l 4:o 5:, 6:空格 7:你 10:好
```

> ⚠️ **注意**：`for range` 遍历**字符串时按 rune（Unicode 码点）迭代**，索引 `i` 是**字节偏移**而非字符序号（中文占 3 字节，所以"你"从 7 开始、"好"从 10 开始）。要"第 N 个字符"不能靠 range 的序号，要先转 `[]rune`。

### 10.3 遍历 map

```go
m := map[string]int{"a": 1, "b": 2, "c": 3}
for k, v := range m { fmt.Printf("%s=%d\n", k, v) }
```

> ⚠️ **注意（重点）**：map 的 `range` 遍历顺序是**随机的**（每次运行可能不同）！这是 Go 故意的设计（防止依赖遍历顺序）。所以不要用 map 遍历顺序做任何假设，需要有序就先取 key 排序（如 `collections.MapKeysSorted`）。

### 10.4 range 的陷阱

```go
nums := []int{1, 2, 3}
for _, v := range nums {
    v *= 2  // 修改的是副本，原切片不变
}
fmt.Println(nums)  // [1 2 3]

for i := range nums {
    nums[i] *= 2  // 通过索引修改
}
fmt.Println(nums)  // [2 4 6]
```

> ⚠️ **注意（经典陷阱）**：`for _, v := range nums` 里的 `v` 是元素的**副本**，改 `v` 不影响原切片。且 range 在循环开始前就**拷贝了原序列的快照**（对 slice 是描述符快照，对 array 是值拷贝）——在循环里 append 改变 slice 长度，通常不影响正在进行的迭代。要改元素用索引 `nums[i]`。

## 11. break 和 continue

### 11.1 break / 11.2 continue

```go
for i := 0; i < 10; i++ {
    if i == 5 { break }
    if i%2 == 0 { continue }
    fmt.Println(i)  // 1 3
}
```

### 11.3 带标签的 break/continue

```go
outer:
    for i := 0; i < 10; i++ {
        for j := 0; j < 10; j++ {
            if i*j > 50 { break outer }
        }
    }
```

> ⚠️ **注意**：多层循环里 `break`/`continue` 默认只作用于**最内层**循环。要跳出外层，用**标签**（label）+ `break outer` / `continue outer`。Go 的标签用法比 C 的 goto 受限得多，只能配合 break/continue，不能任意跳转。

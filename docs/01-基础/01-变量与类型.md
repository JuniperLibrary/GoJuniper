# 01 — 变量与类型

## 是什么

Go 是静态类型语言，变量声明时需指定类型（或通过短声明让编译器推导）。类型系统简单且偏显式，没有隐式类型转换。

> ⚠️ **注意**：Go 的"可见性"靠**标识符首字母大小写**决定，不是 `public`/`private` 关键字。大写 = 导出（public），小写 = 包内私有。这和 Java 的可见性标注（`public`/`private`/`protected`/包私有）思路不同（Java 用关键字，Go 用命名约定），是初学者最常忽略的点。

## 怎么用

### 变量声明

```go
// 短声明（推荐，只能用于函数内）
a := 42

// 显式类型声明
var b int = 42

// 零值初始化（int 默认 0，string 默认 ""，bool 默认 false）
var c int
```

> ⚠️ **注意**：`var m map[string]int` 声明的 map 零值是 `nil`，**直接 `m["k"]=1` 会 panic**。必须用 `make(map[string]int)` 初始化后才能写入。slice/chan 同理，只有 make/new 之后才是可用的非空值。

### 基础类型

| 类型 | 说明 | Java 对照 |
|------|------|-----------|
| `int` / `uint` | 平台字长（32位/64位） | `int`（固定 32 位）/ `long`（64 位） |
| `int8`..`int64` | 定长有符号 | `byte` / `short` / `int` / `long` |
| `uint8` (byte) | 字节 | `byte` |
| `rune` (= `int32`) | Unicode 码点 | `char`（UTF-16 码元） |
| `float64` / `float32` | 浮点 | `double` / `float` |
| `bool` | 布尔 | `boolean` |
| `string` | 不可变字节序列（UTF-8） | `String`（UTF-16） |
| `error` | 错误接口 | `Exception` / `Throwable` |

> ⚠️ **注意**：`int` 的**宽度随平台变化**（32 位 = 4 字节，64 位 = 8 字节）。需要定长时务必显式写 `int64` / `uint64`，否则跨平台数值可能溢出或不一致。

### 控制流

```go
// for 是 Go 唯一的循环关键字
for i := 0; i < n; i++ { }     // 带初始化的 for
for i < n { }                   // 相当于 while
for range xs { }                // 相当于 for _ in &xs
for i, v := range xs { }        // 相当于 for (i, v) in xs.iter().enumerate()

// switch 不需要 break，匹配后自动退出
switch i % 15 {
case 0: // 15 的倍数
    out = append(out, "FizzBuzz")
case 0:                          // 编译错误：重复 case
}
```

参见 `internal/01-basics/basics.go:45` 的 `FizzBuzz` 实现。

### 切片（slince）与字符串

```go
// 切片是动态数组，类似 Java ArrayList
xs := []int{3, 1, 10, -2}
xs = append(xs, 7)

// string 不可变，需要时转 []rune 处理 Unicode
rs := []rune("你好Go")    // [你, 好, G, o]
rs[0], rs[3] = rs[3], rs[0]
s := string(rs)           // "o好G你"
```

参见 `internal/01-basics/basics.go:128` 的 `ReverseString` — 用 `[]rune` 反转字符串，避免拆坏 UTF-8 字节。

## 为什么（对比 Java）

- **零值**：Go 变量隐式初始化为零值（`int` → `0`，`string` → `""`），Java 的类字段也有默认零值（0/`false`/`null`），但局部变量必须显式初始化否则编译报错。好处是简化代码，坏处是可能遗忘真正的初始化意图——尤其是 nil map/slice 写入 panic 的坑。
- **无 `const` 泛型**：Go 的 const 只能是数字/字符串/布尔，没有 Java 的编译期常量表达式那种复杂计算能力。
- **多返回值替代 `Result`**：Go 用 `(value, ok)` / `(value, error)` 二值模式，Java 用 `Optional<T>` / `try-catch`/`Exception` 表达可选与错误。Go 的模式更"原始"，但足够直接。
- **`for` 闭包变量捕获陷阱**：Go 的 `for i, v := range xs` 中 `i` 和 `v` 每次循环是**同一个变量**（地址不变）。Go 1.22 改进了这个行为，但理解捕获语义仍重要。

> ⚠️ **注意**：Go 存在**未类型常量**（如 `const X = 100`）和**类型化常量**（如 `const X int = 100`）。未类型常量赋值时会按上下文自动适配类型（可赋给 int/int64/float64），非常灵活；类型化常量则不能隐式转换到其他类型，更安全。需要灵活度用未类型，需要安全边界用类型化。

## 常见坑

1. **`int` 在 32 位平台是 4 字节** — 跨平台要注意。需要定长时用 `int64` / `uint64`。
2. **`string` 索引访问的是字节，不是字符** — `s[0]` 拿到的是 `byte`。遍历字符用 `for i, r := range s`。
3. **切片是引用类型，赋值不拷贝数据** — `a := xs; a[0] = 100` 会修改原切片。需要独立副本用 `copy()`。
4. **`switch` 的 `case` 默认 `break`** — 不需要显式写 `break`，这和 Java 的 `switch`（传统 fall-through 需 break）不同。需要穿透时用 `fallthrough`（很少用）。
5. **`for range` 的 key 只有 map/channel 有意义** — 对 slice 的 `for i, v := range xs`，`i` 是索引，`v` 是值拷贝。Java 的 `for (int i = 0; i < xs.size(); i++)` 或 `IntStream.range().forEach()` 模式类似。
6. **`uint64` 溢出静默绕回** — Go 不会 panic，需要用 `math.MaxUint64` 手动检查。见 `FactorialUint64` 的溢出检测（`basics.go:92`）。
7. **`string` 不可变** — 任何"修改"都要重新构造（`[]rune` → 改 → `string()`），不能直接原地改字符。

# 02 — 函数进阶

## 是什么

Go 函数是一等公民：可作为参数、返回值，可赋值给变量。闭包、defer、变参、panic/recover 是 Go 函数进阶的核心特性。

## 怎么用

### 闭包（closure）

闭包捕获外层变量，类似 Rust 的 `|| { }` 语法：

```go
func Counter() func() int {
    var count int          // 被闭包捕获
    return func() int {
        count++            // 修改捕获的变量
        return count
    }
}

c := Counter()
c() // 1
c() // 2
// 每个 Counter() 调用创建独立的闭包状态
```

参见 `internal/funcsx/funcsx.go:12` 的 `Counter` 实现。

### defer — 延迟执行

defer 在函数返回前执行，后进先出（LIFO）：

```go
func DeferInspect(nums []int) (log string) {
    for _, n := range nums {
        defer func() {
            log += fmt.Sprintf("defer(%d)", n)
        }()
    }
    log += "body"
    return
}
// 结果："bodydefer(3)defer(2)defer(1)"
```

参见 `internal/funcsx/funcsx.go:44`。

陷阱：**defer 的参数立即求值**，但闭包引用的变量延迟求值。上述代码用闭包捕获 `n`，避免了 `defer log += fmt.Sprintf("defer(%d)", n)` 立即求值的问题。

### 变参函数（variadic）

```go
func Sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

Sum(1, 2, 3)          // 多个参数
Sum(nums...)          // 展开切片
Sum()                 // 空调用 → 0
```

参见 `internal/funcsx/funcsx.go:33`。

### panic / recover

panic 类似 Rust `panic!`，recover 类似 `catch_unwind`：

```go
func RecoverFromPanic(f func()) (err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("panicked: %v", r)
        }
    }()
    f()
    return nil
}
```

参见 `internal/funcsx/funcsx.go:22`。

## 为什么（对比 Rust）

- **Go 闭包默认可变捕获**：Rust 闭包需 `move` / `&` / `&mut` 关键字标明捕获方式；Go 闭包默认捕获变量的引用，无需额外标记。Go 没有生命周期标注，所以闭包用起来"更省事"，但无法在编译期防止悬垂引用。
- **defer ≈ Rust Drop**：defer 是作用域结束前的显式清理，Rust 用 RAII 的 `Drop` trait 自动清理。Go 的 defer 需要手写，但更灵活（比如能修改返回值）。
- **defer 在函数作用域结束才执行**，不像 Rust Drop 在变量离开作用域时（块级）执行。Go 没有块级 defer。
- **没有 `?` 运算符**：Go 用 `if err != nil` 手动传播错误。函数调用链长时显得冗长，但显式。
- **panic/recover 仅用于"不可能发生的"错误**，类似 Rust 中 `unwrap()` 的预期。正常错误用 `error` 返回值。

## 常见坑

1. **defer 参数立即求值，闭包变量延迟求值**：
   ```go
   // 错误：defer 时已求值 x=0，最终全是 0
   for x := 0; x < 3; x++ {
       defer fmt.Print(x)
   }
   // 正确：闭包捕获 x，发现 2,1,0（LIFO）
   for x := 0; x < 3; x++ {
       defer func() { fmt.Print(x) }()
   }
   ```
   Go 1.22 后 `for` 循环变量重建，上述问题已缓解。但仍需理解 defer 求值时机。

2. **recover 只能在 defer 中有效** — 在 `defer` 函数外调用 `recover()` 返回 `nil`。

3. **panic 会越过所有 defer 执行 recover** — 如果 defer 中的 recover 没有命中，panic 继续向上传播。

4. **`for` 中 defer 累积不会在每次迭代后执行** — 所有 defer 在包含它们的函数返回前才执行。循环中大量使用 defer 会导致资源延迟释放（defer 栈膨胀）。

5. **变参展开切片时不能传 `[]int` 本身** — 需要加 `...`：`Sum(nums...)`。

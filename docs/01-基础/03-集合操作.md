# 03 — 集合操作

## 是什么

Go 提供两种内置集合类型：**slice**（动态数组）和 **map**（哈希表）。没有 Rust 的 `Vec` / `HashMap` 标准库类型，它们直接内嵌在语言中。slice 和 map 都是引用类型，传递时共享底层数据。

## 怎么用

### 切片（slice）构造

```go
// 字面量
xs := []int{3, 1, 10, -2}

// make 预分配（指定长度和容量）
xs := make([]int, 0, n)   // len=0, cap=n，后续 append 不触发扩容

// 切片表达式（不拷贝数据）
sub := xs[1:3]            // xs 的子视图
```

### 用 map 去重和计数

```go
// 去重：用 map[int]struct{} 做 set
func UniqueInts(xs []int) []int {
    seen := make(map[int]struct{}, len(xs))
    out := make([]int, 0, len(xs))
    for _, v := range xs {
        if _, ok := seen[v]; ok {
            continue          // 已存在，跳过
        }
        seen[v] = struct{}{}
        out = append(out, v)
    }
    return out
}
// UniqueInts([]int{3, 3, 1, 2, 1, 2, 4}) → [3, 1, 2, 4]（保持首次出现顺序）
```

```go
// 计数：map 的零值是 0，可以直接自增
func Frequency(xs []string) map[string]int {
    out := make(map[string]int, len(xs))
    for _, s := range xs {
        out[s]++    // 首次访问时 out[s] == 0，自增后为 1
    }
    return out
}
// Frequency([]string{"a","b","a","c","b","a"}) → {"a":3, "b":2, "c":1}
```

参见 `internal/collections/collections.go:10` 的 `UniqueInts` 和 `collections.go:24` 的 `Frequency`。

### map key 排序 + 泛型

Go map 的遍历顺序是**随机**的，不能依赖。需要稳定顺序时先提取 key 再排序：

```go
func MapKeysSorted[V any](m map[string]V) []string {
    keys := make([]string, 0, len(m))
    for k := range m {
        keys = append(keys, k)
    }
    sort.Strings(keys)
    return keys
}
```

`[V any]` 是 Go 1.18+ 的泛型语法，对应 Rust 的 `<V>`（无 trait bound）。参见 `internal/collections/collections.go:34`。

## 为什么（对比 Rust）

- **slice 没有所有权语义**：Go 的 slice 任何赋值都是"引用传递"（拷贝 `ptr/len/cap` 三个字段），没有 borrow checker。这让 slice 的传递成本很低，但需要开发者注意共享修改问题。
- **无 `mut` 概念**：Go 没有独占可变引用概念，并发时需靠 `sync.Mutex` 或 channel 人为保证安全。
- **map 无 entry API**：Rust 的 `HashMap::entry()` 可以避免两次哈希，Go 只能先读再写（或直接自增，利用零值特性）。
- **`struct{}` 做 set**：Go 没有内置 set 类型，用 `map[K]struct{}` 模拟。`struct{}` 不占内存（`unsafe.Sizeof(struct{}{}) == 0`），是最节省的表达。
- **`make` 预分配容量**：`make([]T, 0, n)` 预先分配容量，避免 append 多次扩容拷贝。Rust 的 `Vec::with_capacity(n)` 同理。

## 常见坑

1. **map 的零值是 `nil`，写入 panic**：声明 `var m map[string]int` 后直接写 `m["key"] = 1` 会 panic。必须用 `make` 初始化。
   ```go
   var m map[string]int       // nil
   m["a"] = 1                 // panic: assignment to entry in nil map
   m = make(map[string]int)   // 必须先初始化
   ```

2. **map 遍历顺序随机**：不要依赖 map 的遍历顺序。需要有序输出时，提取 key 排序。

3. **slice 扩容 ≠ in-place**：`append` 返回新 slice，原 slice 容量不足时会分配新底层数组。调用方必须接收返回值：
   ```go
   xs = append(xs, v)   // 必须重新赋值，否则容量不足时丢失数据
   ```

4. **`struct{}` 是 set 的惯用实现**：不要用 `map[K]bool` 作为 set，浪费 1 字节。用 `map[K]struct{}`，判断存在性用 if _, ok := m[K]; ok。

5. **泛型 `[V any]` 无类型约束**：Go 的泛型约束用 `interface` 表达，`any` 是 `interface{}` 的别名，表示任意类型，对应 Rust 的 `V: ?Sized`（但意义不同）。

6. **切片表达式不拷贝数据**：`sub := xs[1:3]` 共享同一底层数组。修改 `sub[0]` 会反映到 `xs`。需要独立数据时用 `copy`。

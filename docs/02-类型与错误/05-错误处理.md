# 05 — 错误处理

## 是什么

Go 的错误处理基于**显式返回**：函数返回 `error` 接口。
没有 `try/catch` 或异常（exception）。error 是一个内置接口：

```go
type error interface {
    Error() string
}
```

## 怎么用

### 哨兵错误

用 `errors.New` 定义包级别的错误变量，调用方用 `errors.Is` 判断。

```go
var ErrNotPositive = errors.New("value must be positive")

func ParsePositiveInt(s string) (int, error) {
    n, err := strconv.Atoi(s)
    if err != nil {
        return 0, fmt.Errorf("parse int: %w", err)
    }
    if n <= 0 {
        return 0, ErrNotPositive
    }
    return n, nil
}
```

调用方：

```go
_, err := ParsePositiveInt("0")
if errors.Is(err, ErrNotPositive) {
    // 处理"不是正数"的错误
}
```

> ⚠️ **注意**：判断错误类型**必须用 `errors.Is` / `errors.As`，不要用 `==` 比较**。因为 `%w` 包裹后的错误是包装链，`==` 比较会失败，而 `errors.Is` 会顺着链一路匹配到底层哨兵错误。

### 包裹错误（%w）

用 `fmt.Errorf("%w", err)` 包裹底层错误，保留原始错误链，
调用方仍然可以用 `errors.Is` 或 `errors.As` 解开。

```go
n, err := strconv.Atoi(s)
if err != nil {
    return 0, fmt.Errorf("parse int: %w", err)
}
```

`%w` 和 `%v` 的区别：

- `%w`：记录错误链，`errors.Is` 可以穿透
- `%v`：只保留字符串信息，丢失原始错误类型

> ⚠️ **注意**：**用 `%w` 才能保留错误链**；如果用 `%v` 或 `%s` 包裹，原始错误类型会丢失，`errors.Is` 再也匹配不到底层哨兵错误。所以"需要链式识别"的错误一定要用 `%w`。

### 合并错误

`errors.Join` 可以将多个 error 合并成一个。

```go
func Join(a, b error) error {
    if a == nil {
        return b
    }
    if b == nil {
        return a
    }
    return errors.Join(a, b)
}
```

> ⚠️ **注意**：自定义错误类型想被 `errors.As` 识别，需让该类型实现 `error` 接口（即提供 `Error() string` 方法）。本仓库的 `DivideError` 就是示例：通过实现 `Error()` 让 `*DivideError` 成为标准错误，可用 `errors.As(err, &var)` 提取具体类型。

## 为什么

对比 Java：

| 特性 | Go | Java |
|------|----|------|
| 错误范式 | 显式返回值 `(T, error)` | 异常 `throw` / `try-catch` / `Optional<T>` |
| 传播方式 | `if err != nil { return err }` | `throw` 自动向上传播 |
| 自定义错误 | 实现 `error` 接口 | 继承 `Exception` / `RuntimeException` |
| 类型匹配 | `errors.Is` / `errors.As` | `catch (SpecificException e)` / `instanceof` |
| 错误链 | `fmt.Errorf("%w", ...)` | `initCause()` / `getCause()`（异常链） |

核心区别：

- Java 用异常机制让错误传播更简洁（自动沿调用栈上抛），Go 需要手写 `if err != nil`。
- Go 的 `%w` 包裹在标准库中内建了错误链支持，Java 用异常链（`cause`）。
- Java 的 `catch` 分支让错误类型匹配更精确，Go 的 `errors.Is` 是线性遍历链。

## 常见坑

1. **哨兵错误用值比较而非 `errors.Is`**：
   不要用 `==` 比较 `error`，要用 `errors.Is`。
   因为 `%w` 包裹后的错误 `==` 会失败，`errors.Is` 会遍历整条链。

2. **只会写 `if err != nil { return err }`**：
   直接返回原始错误会丢失上下文。推荐用 `fmt.Errorf("xxx: %w", err)`。

3. **忽略错误**：
   用 `_` 丢弃错误是合法的，但通常是个坏习惯——至少用 `//nolint` 显式标记。

4. **错误变量的命名**：
   前缀用 `Err`（包级导出）或 `err`（包级私有），如 `ErrInvalidID`。

5. **`errors.Is` 与自定义错误**：提取具体错误类型用 `errors.As(err, &target)`，而非类型断言——断言拿不到被 `%w` 包裹的底层错误。

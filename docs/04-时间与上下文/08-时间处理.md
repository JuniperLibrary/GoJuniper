# 08 — 时间处理

## 是什么

Go 标准库 `time` 提供时间表示（`time.Time`）、时间间隔（`time.Duration`）、
格式化/解析、时区、定时器等核心能力。一个包覆盖日常 90% 的时间需求。

## 怎么用

### 解析时间字符串

```go
// 解析 RFC3339/RFC3339Nano 格式
t, err := time.Parse(time.RFC3339Nano, "2026-03-11T12:34:56Z")
```

`time.Parse` 使用布局字符串（layout）而非格式指令（如 `%Y-%m-%d`）。
Go 的布局字符串基于一个固定参考时间：`Mon Jan 2 15:04:05 MST 2006`
（按顺序：月 日 时 分 秒 年 时区）。

```go
// 自定义格式：参考时间是 01/02 03:04:05PM '06 -0700
t, _ = time.Parse("2006-01-02 15:04:05", "2026-03-11 12:34:56")
```

### Duration 计算

```go
func Until(deadline, now time.Time) time.Duration {
    return deadline.Sub(now)  // 可能为负数
}
```

`time.Duration` 本质是 `int64`，单位为纳秒。有预定义常量方便使用：

```go
time.Second          // 1 秒
time.Millisecond     // 1 毫秒
100 * time.Millisecond // 100 毫秒
```

### 获取某天 00:00:00（业务常见需求）

```go
func StartOfDay(t time.Time) time.Time {
    y, m, d := t.Date()
    return time.Date(y, m, d, 0, 0, 0, 0, t.Location())
}
```

`time.Date` 接受年、月、日、时、分、秒、纳秒、Location，返回新的 `Time`。
关键：`t.Location()` 保留了原时区信息，结果仍然是对应的时区。

### 关键设计点

- **Location（时区）**：`time.Time` 内部包含时区信息，`Equal` 方法比较时会忽略时区差异（都转 UTC 后再比）
- **Monotonic clock**：`Since`、`Until` 等操作使用单调时钟，不受系统时间调整影响
- **Timer / Ticker**：`time.NewTimer(d)` 返回 `*time.Timer{ C chan Time }`，用于超时控制

## 为什么

对比 Rust：

| 概念 | Go | Rust |
|------|----|------|
| 时间点 | `time.Time` | `chrono::NaiveDateTime` / `DateTime<Tz>` |
| 时间间隔 | `time.Duration`（int64 纳秒） | `chrono::Duration` 或 `std::time::Duration` |
| 格式/解析 | layout-based 字符串 | `chrono::format` 与 `strftime` 指令 |
| 时区 | `time.Location` 指针 | `chrono::FixedOffset` / `Utc` / `Local` |
| 定时器 | `time.Timer`, `time.Ticker` | `tokio::time::sleep`（异步）|

Go 的 layout 字符串是个争议设计。它的灵感是"用示例描述格式"而非用指令组合，
初学不习惯，但熟悉参考时间 `Mon Jan 2 15:04:05 MST 2006` 后就自然了。

Rust 的 `chrono` 用 `%Y-%m-%d` 风格的格式指令，更接近 C 传统。

## 常见坑

1. **`time.Parse` 用 layout，不是 format string**：
   写 `"2006-01-02"` 而不是 `"%Y-%m-%d"`。忘记参考时间顺序是最常见的错误。

2. **`==` vs `Equal`**：
   `==` 比较 `time.Time` 时会比较时区信息，即使表示同一时刻也可能不相等。
   用 `t.Equal(u)` 方法，它忽略时区差异。

3. **`time.Now()` 的 Location**：
   `time.Now()` 返回系统本地时间（含时区）。UTC 获取用 `time.Now().UTC()`。

4. **序列化默认格式**：
   `time.Time` 的 JSON 默认输出是 `"2006-01-02T15:04:05Z07:00"`（RFC3339）。
   很多数据库和 API 也期望这种格式，不要擅自改成自定义格式。

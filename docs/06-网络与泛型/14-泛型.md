# 14 — 泛型

## 是什么

Go 1.18（2022 年）引入的泛型允许函数和类型通过**类型参数**（type parameter）对多种类型复用。语法上使用方括号 `[T any]` 而不是尖括号 `<T>`。约束（constraint）用接口表达，`any` 是 `interface{}` 的别名。

## 怎么用

### 类型参数和约束

```go
// Map 把 []A 映射为 []B，A B 可以是任意不同类型
func Map[A any, B any](xs []A, f func(A) B) []B {
    out := make([]B, 0, len(xs))
    for _, x := range xs {
        out = append(out, f(x))
    }
    return out
}
```

### 常见的泛型高阶函数

```go
// Filter 保留满足条件的元素
func Filter[T any](xs []T, pred func(T) bool) []T {
    out := make([]T, 0, len(xs))
    for _, x := range xs {
        if pred(x) {
            out = append(out, x)
        }
    }
    return out
}

// Reduce 归并
func Reduce[T any, Acc any](xs []T, acc Acc, f func(Acc, T) Acc) Acc {
    for _, x := range xs {
        acc = f(acc, x)
    }
    return acc
}
```

### 带约束的泛型：`cmp.Ordered`

```go
func GetLargest[T cmp.Ordered](xs []T) (T, bool) {
    if len(xs) == 0 {
        var zero T
        return zero, false  // Go 没有 Option，用 bool 表示存在性
    }
    largest := xs[0]
    for _, v := range xs[1:] {
        if v > largest {
            largest = v
        }
    }
    return largest, true
}
```

`cmp.Ordered` 是 Go 1.21 新增的内置约束，覆盖所有支持 `< <= > >=` 的类型（int、float64、string、rune 等）。试用：

```go
GetLargest([]int{34, 50, 25, 100, 65})        // (100, true)
GetLargest([]rune{'y', 'm', 'a', 'q'})         // ('y', true)
GetLargest([]float64{3.14, 2.71, 1.618})       // (3.14, true)
```

## 为什么

对比 Rust 的泛型：

| 方面 | Rust | Go |
|------|------|----|
| 引入时间 | 1.0（2015） | 1.18（2022） |
| 语法 | `<T: Trait>` | `[T constraint]` |
| trait bound vs 接口约束 | `T: PartialOrd` | `T cmp.Ordered` |
| 零值表达 | `None` / `std::mem::zeroed()` | `var zero T` |
| 单位类型 | `()` | 无（返回 `(T, bool)` 模拟 Option）|
| 类型参数方法 | 支持 `impl<T> Foo<T>` | 支持（1.18+） |
| 特化（specialization） | 支持 | 不支持 |

核心差异在于 Rust 的泛型是**编译期单态化 + trait bound 系统**，Go 的泛型也是单态化，但约束系统等价于**接口**，表达能力比 Rust 的 trait 弱：

- Go 没有关联类型（associated type），没有生命周期参数，没有 `where` 从句的复杂条件
- Go 的约束必须是接口（或 `any`），不能像 Rust 那样 `T: Display + Clone` 的组合约束
- Go 的运算符 `>` 不通过 trait 表达——只有编译器内置类型支持，自定义类型不能重载运算符。`cmp.Ordered` 只是一个标记接口，不是让用户实现 `Ord` trait 的方式

## 常见坑

1. **泛型不能用于方法接收者的类型参数**（1.18 限制）：

```go
// 编译错误：方法不能有额外的类型参数
func (s *MySlice[T]) Map[U any](f func(T) U) []U { ... }
```

2. **`any` 不是无约束**：`[T any]` 等价于 `[T interface{}]`，可以传任意类型，但无法使用 `>`、`+` 等运算符，只能调用 `T` 接口上声明的方法。

3. **没有 `Option<T>`**：Go 惯用 `(T, bool)` 或指针 + nil 表达可选值，不支持 Rust 的 `Option<T>` 模式匹配。

4. **性能认知**：Go 泛型是单态化的，运行时性能与手写特定类型版本一致，但编译产物会膨胀（代码膨胀问题类似 C++ template）。
